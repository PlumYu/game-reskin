日期：2026-05-27

## 1. 范围

本次只分析 `E:\project\game\game_nmxxq\B` 这个 Cocos Creator 工程。

当前阶段只制定计划，不替换具体素材，不改玩法逻辑，不改关卡规则。

后续换皮目标是：保持原有方格观察玩法不变，只替换主题包装、目标物、美术资源、展示文案和 OSS 资源路径。

## 2. 原玩法边界

底层玩法保持不变：

- 玩家在方格里观察隐藏目标。
- 点击打叉排除干扰格。
- 双击标记找到的目标。
- 关卡配置控制格子数量、颜色区域和目标位置。

换皮时不能改成跑酷、经营、收集、推理剧情或其他新玩法。

## 3. 当前资源结构

### 3.1 远程资源源目录

路径：

`B/assets/RemoteBundle`

这里的资源会被 Cocos 打进 `remote` bundle。微信构建后，脚本会把它移动到：

`B/build/cdn-bundle`

之后再上传到阿里云 OSS。

主要内容：

- `images/`：首页背景、标题、按钮、提示图、HUD 图片。
- `balloon_atlas/`：棋盘格子/气球视觉素材。
- `characters/`：首页角色静态图与动画帧。
- `configs/`：主线、生存、每日挑战关卡 JSON。
- `guide/`：引导相关图片。
- `effects/`：结算或奖励特效图片。
- `audio/`：BGM 和音效。

### 3.2 首屏兜底资源目录

路径：

`B/assets/BootstrapBundle`

这里是首屏和兜底资源。部分资源会优先从 `bootstrap` bundle 加载，如果找不到再回退到 `remote` bundle。

换皮时如果只改 `RemoteBundle`，首屏可能短时间仍然出现旧主题，所以首屏相关资源需要同步检查 `BootstrapBundle`。

### 3.3 历史本地资源目录

路径：

`B/assets/images`

这里存在一些历史图片资源。后续换皮前需要确认是否仍被场景节点或旧代码直接引用。当前优先级低于 `RemoteBundle` 和 `BootstrapBundle`。

## 5. OSS 与构建链路

相关脚本位于原游戏仓库：

- `scripts/fix_wechat_build.js`
- `scripts/sync_cdn_oss.js`
- `scripts/build_wechat.js`

默认 OSS 配置：

- bucket：`game-nmxxq`
- 默认 prefix：`nmxxq/wechat/`
- 默认 CDN：`https://game-nmxxq.oss-cn-beijing.aliyuncs.com/nmxxq/wechat/`

重要风险：

多个换皮包不能共用同一个 OSS prefix。否则后上传的包会覆盖前一个包的远程资源。

建议每个包使用独立 OSS prefix：

- `nmxxq/wechat/laoban/`
- `nmxxq/wechat/naicha/`
- `nmxxq/wechat/huoguo/`
- `nmxxq/wechat/kuaidi/`

构建上传时需要同时保持这两个配置一致：

- `NMXXQ_OSS_PREFIX`
- `NMXXQ_CDN_BASE_URL`

## 6. 换皮执行计划

### 阶段 1：确认目标包

每次只做一个包，不同时改 10 个。

需要先确认：

- 游戏主题名称。
- 平台申请名称。
- 目标物是什么。
- 方格代表什么。
- 是否需要改首页背景、标题、角色、提示图、格子图。
- OSS prefix。

### 阶段 2：建立素材替换清单

优先检查并替换：

- `B/assets/RemoteBundle/images/menu_main_bg.png`
- `B/assets/RemoteBundle/images/menu_title_logo.png`
- `B/assets/RemoteBundle/images/提示-找牛.png`
- `B/assets/RemoteBundle/images/提示-排除.png`
- `B/assets/RemoteBundle/characters/menu_idle_static.png`
- `B/assets/RemoteBundle/characters/menu_idle/*.png`
- `B/assets/RemoteBundle/balloon_atlas/*.png`

同步检查：

- `B/assets/BootstrapBundle/images/背景图.png`
- `B/assets/BootstrapBundle/characters/menu_idle_static.png`
- `B/assets/BootstrapBundle/balloon_atlas/*.png`

### 阶段 3：本地替换与预览

替换素材时保留 Cocos `.meta` 文件，不随意删除目录。

替换后在 Cocos Creator 打开 `B` 工程预览，重点检查：

- 首页是否仍出现旧牛主题。
- 进入关卡后目标物是否统一。
- 提示按钮是否还是旧“找牛”表达。
- 首屏加载是否出现旧图。
- 方格、叉号、双击标记流程是否未变化。

### 阶段 4：构建与 OSS 上传

先本地构建，不直接上传 OSS。

确认构建产物：

- `B/build/wechatgame`
- `B/build/cdn-bundle`

确认无误后，再使用独立 prefix 上传。

示例：

```powershell
$env:NMXXQ_OSS_PREFIX='nmxxq/wechat/laoban/'
$env:NMXXQ_CDN_BASE_URL='https://game-nmxxq.oss-cn-beijing.aliyuncs.com/nmxxq/wechat/laoban/'
node scripts/build_wechat.js
```

如果只想先构建，不上传：

```powershell
$env:NMXXQ_SKIP_CDN_SYNC='1'
node scripts/build_wechat.js
```

### 阶段 5：微信开发者工具验证

验证点：

- 小游戏能正常启动。
- 远程资源能加载。
- 首屏、首页、关卡、结算页无旧主题穿帮。
- 不同包的 OSS 资源不会互相覆盖。
- 手机真机能进入关卡。
